# OlymRoad
Платформа олимпиадного пути ученика (`Roadmap + Tracking + Analytics`).

## 1. Название проекта
`OlymRoad` — платформа олимпиадного пути ученика.

## 2. Цель и идея
Создать веб/мобайл-приложение, где ученик видит единый график/календарь олимпиад как дорожную карту и может:
- записываться на олимпиады и этапы;
- планировать подготовку;
- фиксировать результаты;
- отслеживать прогресс по направлениям;
- получать аналитику (сильные/слабые темы, динамика, прогноз).

## 3. Целевая аудитория и роли
- `Ученик` — строит путь, готовится, отмечает активность и результаты.
- `Тренер/учитель` — ведет группу/учеников, ставит планы, анализирует.
- `Администратор/организатор` — управляет базой олимпиад/этапов, правами, справочниками.

## 4. Основные сущности (данные)
- `Олимпиада`: название, предмет/направление, уровень (школьный/район/область/республиканский/международный), формат, ссылки, организатор, правила.
- `Сезон/Год`: `2025-2026` и т.д.
- `Этап`: дата(ы), регистрация (дедлайн), место/онлайн, требования.
- `Профиль ученика`: класс, школа, направления, цели (например, "выйти на областной этап").
- `Активности подготовки`: темы, задачи, тренировки, пробники, время, материалы.
- `Результат`: баллы, место, статус (участник/призер/победитель), комментарии.
- `Навыки/Темы` (опционально): по предмету (алгебра/геометрия/механика и т.д.).

## 5. Пользовательские сценарии (User Stories)
### Ученик
- Видит roadmap олимпиад по выбранному направлению на год.
- Фильтрует по предмету/уровню/месяцу/статусу регистрации.
- Нажимает "Записаться" и получает карточку этапа + чек-лист.
- Ведет подготовку: отмечает тренировки, пробники, темы.
- Вносит результаты после этапа.
- Смотрит графики прогресса и зоны проседания.

### Учитель/тренер
- Создает группу (например, "Физика 9 класс").
- Добавляет учеников, назначает планы подготовки.
- Видит сводку по группе: кто зарегистрирован, кто пропустил дедлайн, динамика пробников.
- Экспортирует отчет (`PDF/Excel` — опция).

### Админ
- Добавляет/редактирует олимпиады и этапы, подтверждает данные.
- Управляет справочниками (предметы, уровни, регионы).
- Управляет доступами/ролями.

## 6. Функциональные требования (MVP)
### 6.1 Roadmap/календарь олимпиад
- Годовой/месячный календарь.
- Лента этапов (список) + карточка события.
- Фильтры: предмет, уровень, формат, месяц, "скоро дедлайн".
- Статусы: "в планах", "зарегистрирован", "участвовал", "результат внесен".

### 6.2 Запись и дедлайны
- Кнопка "Записаться" внутри приложения.
- Сохранение дедлайнов и внутренние уведомления.
- Чек-лист подготовки: документы/согласия/взнос/платформа.

### 6.3 Трекер подготовки
- Дневник занятий (дата, длительность, тема, тип: задачи/теория/пробник).
- Цели недели/месяца (количество задач/часов/пробников).
- Прикрепление ссылок/материалов.

### 6.4 Результаты и аналитика
- Ввод результатов (баллы/место/уровень/этап).
- Графики:
- прогресс баллов по времени;
- активность подготовки (часы/задачи);
- "воронка" по этапам (сколько дошел);
- сравнение "подготовка ↔ результат" (простая корреляция).
- Рекомендации (минимально): "мало пробников" / "не закрыта тема X".

### 6.5 Кабинет учителя
- Список учеников и статусы по ближайшим этапам.
- Общая панель: дедлайны, регистрации, активность, результаты.
- Комментарий учителя к ученику (текст).

## 7. Расширение (Nice-to-have)
- Импорт календаря (`.ics`) / синхронизация с `Google/Apple` (опция).
- Лента достижений + бейджи.
- Генератор индивидуального плана подготовки на `4-8` недель.
- Рейтинг внутри школы/группы (если разрешено политиками).
- AI-помощник по подбору задач/тем (если допускается).

## 8. Нефункциональные требования
- `UX`: быстро, не более 3 кликов до "записаться/посмотреть дедлайн".
- `Производительность`: открытие дашборда `<= 2 сек` на среднем ноутбуке.
- `Безопасность`: роли, доступ только к своим данным/группам.
- `Локализация`: минимум `RU/KZ` для интерфейса.
- `Доступность`: мобильная адаптация обязательна.

## 9. Ограничения хакатона
- Время разработки: `24 часа`.
- Можно использовать мок-данные олимпиад (`seed`) или упрощенную БД.
- Уведомления: внутри приложения (`push/email` — опционально).

## 10. Технологические ожидания (не жестко)
- `Frontend`: `React` / `Next.js` / `Flutter` (если мобильное).
- `Backend`: `Node.js` / `Python (FastAPI)` / `Firebase`.
- `DB`: `PostgreSQL` / `Firestore`.
- `Charts`: `Chart.js` / `Recharts` / `ECharts`.

## 11. Интеграции (по желанию)
- Авторизация: `Google` / `Microsoft` / `Email`.
- Экспорт: `PDF/Excel`.
- Внешние ссылки на страницы олимпиад (без парсинга).

## 12. Структура интерфейса (минимальный набор экранов)
- Онбординг (выбор направления, цели, класс).
- Roadmap (календарь + список).
- Карточка олимпиады/этапа (даты, дедлайн, чек-лист, запись).
- Трекер подготовки (добавить занятие/пробник).
- Аналитика (графики прогресса).
- Кабинет учителя (группа, сводка, ученик).

## 13. Данные для демо (seed)
Подготовить `10-20` олимпиад/этапов (примерно на год) по `2-3` предметам и показать:
- `1` ученика с активностью;
- `1` учителя с группой;
- `2-3` результата и графики.

## 14. Критерии приемки (Definition of Done)
MVP считается готовым, если:
- есть roadmap с фильтрами и карточками этапов;
- ученик может "записаться", видеть дедлайн и статус;
- есть трекер подготовки (минимум: добавить занятие);
- можно внести результат и увидеть минимум 2 графика;
- учитель видит сводку по ученикам (хотя бы статус/ближайшие события);
- есть демо-данные и сценарий демонстрации на `3-5` минут.
